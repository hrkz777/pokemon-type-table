# 現状の設計指針

## 目的

このサイトは、ポケモンのタイプ相性を素早く確認できることを最優先とする。
主目的は装飾ではなく、一覧性・判読性・操作性である。

## 現状のデザイン基準

現状の見た目は、以下の方向を基準としている。

- ポケモン公式のタイプ相性表に近い、情報整理優先のフラットな画面
- 白ベースの面構成
- 赤い見出し帯
- 濃いグレーの軸ラベル
- 細いグリッド線
- タイプ色だけは鮮やかに維持

参考イメージ:

- https://www.pokemon.co.jp/ex/sun_moon/fight/161215_01.html

## ビジュアル方針

### 1. 情報を主役にする

- 背景は目立たせない
- カードやパネルは単色面を基本とする
- 過度なグラデーション、ぼかし、ガラス風表現は使わない
- 装飾よりも、区切り線・余白・位置関係で整理する

### 2. ポケモンらしさはアクセントで出す

- ポケモンらしさは赤・青・黄のアクセントで表現する
- ページ全体を派手にするのではなく、見出し・状態・強調箇所に限定して使う
- タイプ固有色は `types.css` の定義をそのまま尊重する

### 3. タイプ色の可読性を優先する

- タイプバッジは鮮やかな背景色を前提とする
- そのため、文字アウトラインは必須とする
- フラット化を理由に、タイプバッジの文字アウトラインを消してはならない

## レイアウト方針

### 1. 上部構成

- ページ先頭は赤い帯状のヘッダーとする
- その下に白い本体コンテナを置く
- ヘッダーと本体は一体感を持たせるが、過剰な装飾は避ける

### 2. タブ

- タブは本体上部に並べる
- 未選択タブは白または淡色
- 選択中タブは赤で強調する
- タブ自体は読みやすい矩形ベースとし、過度に丸めすぎない

### 3. 相性表

- 相性表はこのサイトの中心要素である
- 上部ヘッダーは赤背景・白文字
- 左側の軸ラベルは濃いグレー背景・白文字
- グリッド線は細く、主張しすぎない
- 交互背景や hover は補助的に使い、色ノイズを増やさない
- sticky header / sticky side column は維持する

### 4. 自分のポケモン用パネル

- タイプ選択と結果表示は、表とは別の補助 UI として扱う
- 相性表より目立たせない
- ただし操作状態は明確にする

## コンポーネント方針

### タイプバッジ

- タイプバッジはこのサイトの視覚的アイコンである
- 色は `types.css` のタイプ別変数を使用する
- 文字アウトラインは維持する
- バッジ内部で別装飾を足しすぎない

### タイプ選択ボタン

- バッジそのものがボタンとして振る舞う構造を維持する
- ボタンの外側に余計な枠や装飾を足さない
- 状態は以下で表現する
	- 未選択: 不透明
	- 選択中: 不透明 + 選択用アウトライン
	- 2つ選択済み時の未選択: 半透明

### 相性カード

- 4倍、2倍、等倍、0.5倍、0.25倍、0倍ごとに分類する
- カードは単色面で差をつける
- グラデーションで立体感を出さない
- 色分けは補助情報であり、文字情報を読めなくしてはならない

## 余白・角丸・影

- 現状はフラット寄りを基本とする
- 角丸は控えめにする
- 影は最小限にする
- 情報テーブル周辺で、強い浮遊感を出さない

## タイポグラフィ方針

- ベースフォントは `Noto Sans JP`
- 見出しは太め、本文は標準
- 文字サイズ差は必要最小限に留める
- 相性表やバッジでは、装飾より可読性を優先する

## 実装上の境界

### CSS

- 全体レイアウトと UI の見た目は `docs/css/styles.css`
- タイプ固有色は `docs/css/types.css`
- タイプごとの配色は `types.css` に集約し、個別インライン style は使わない

### JavaScript

- 画面構築は `docs/js/app.js` が行う
- `index.html` は最小シェルとし、UI の実体は JavaScript で組み立てる
- CSS を前提に DOM を作るため、クラス名の意味を崩さない

### データ

- 非テキストデータは `docs/assets/`
- ローカライズ文言は `docs/locales/<locale>/`
- UI 文言は JavaScript に直接埋め込まない

## 現時点で避けるべきこと

- 意味の薄いグラデーション
- ガラス風 UI
- 過剰な影
- 過剰な丸み
- タイプ色の弱体化
- タイプバッジの文字アウトライン削除
- UI 全体をタイプ色で塗りすぎること
- 参照元と関係の薄い抽象的な Material 風装飾

## 次の再設計で判断基準にすること

- 相性表が最優先で読みやすいか
- タイプバッジの視認性が落ちていないか
- ポケモンらしさが色ノイズではなく整理されたアクセントとして見えているか
- 自分のポケモン用 UI が主役を邪魔していないか
- 装飾ではなく情報整理として正しい変更か
